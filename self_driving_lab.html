Self Driving Lab

Biology lab programmed to function autonomously with specified workflow

My work as a Software Engineer Intern

Project Overview

The Autonomous Discovery Self Driving Laboratories (AD-SDL) project at Argonne National Laboratory aims to automate scientific processes using standard instruments and collaborative robots. The project’s primary focus is on the development, implementation, and commissioning of various instruments to facilitate automated workflows within the Rapid Prototyping Laboratory. This overview outlines the work I accomplished in my 10 week long Software Engineering Internship and the significance of the results.

Project Setup

Initial Setup

The project began with acquiring the 3 instruments used for biology experiments to be programmed, analyzing their current functionality, and constructing three mobile carts to house the machines used in the lab, allowing for flexible and efficient rearrangement as required by different experimental setups.

Instrument Drivers Development

Sealer Driver

One of the key instruments in the project is the Azenta Sealer, an instrument used to provide a seal over microplates. The development team enabled the Sealer to execute specific commands via a serial connection. Using PuTTY for initial testing, the team verified the Sealer’s ability to interpret and respond to commands. A Python program was then developed to facilitate this communication, structuring the driver as a class with functions corresponding to particular commands such as sealing or adjusting temperature.

Although the Sealer encountered a malfunction during testing, preventing further development of feedback interpretation functionality, the foundational work laid the groundwork for future improvements.

Peeler Driver

Developing the driver for the Brooks XPeel, an instrument that removes the seal off of microplates, presented a unique challenge. Initially, the Peeler could only provide feedback but not receive commands. After extensive troubleshooting and research, the issue was identified as the RS232 to USB adapter cable. Replacing it with a direct RS232 to USB A cable resolved the communication problem, allowing the driver to send commands and interpret feedback, including error messages displayed as two-digit codes.

Plate Stacker Driver

The development of the Hudson Plate Stacker driver involved identifying approximately 20 different commands from its Visual Basic code. These commands were used to create a driver that instructed the Plate Stacker to perform specific actions, such as grabbing a plate from a specified stack and placing it on the exchange platform.

Building ROS Structures

Learning and Implementation

With guidance from a graduate student on the project, the development team learned the basics of Robot Operating System (ROS). Key ROS components such as Nodes, Services, and Topics were used to design the ROS structures for the Peeler and Sealer, creating nodes that managed actions and interpreted feedback

For the Peeler, the Description Service contained command information, which the Master Node interpreted and used to send a string to the Action Service. This string corresponded to the user-requested action. The Peeler Node then executed the commands, stored any important feedback, and updated the relevant ROS topics and services.

Conclusion

By the end of the project, the lab's Sealer, Peeler, and Plate Stacker were remotely controllable through the development of Python drivers and ROS structures. This achievement significantly advanced the AD-SDL project’s goal of laboratory automation. The equipment is now part of a cohesive automated system, with potential for future enhancements, including automated error detection using visual data from the integrated camera module.

The project has greatly enhanced the team’s skills and knowledge. Proficiency in Python has improved through daily practice, and the importance of clear code documentation and effective communication has been emphasized. The experience of developing ROS structures has provided a solid foundation for future work in robotics and automation.
